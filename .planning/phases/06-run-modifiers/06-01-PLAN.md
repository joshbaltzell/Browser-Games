# Phase 06 — Run Modifier Cards — Plan 06-01

## Goal

Insert a modifier-selection step between the player clicking "Play"/"Play again" and the
game actually starting. The player picks one of four cards (Glass Cannon, Headstart,
Bullet Hell, Standard Run) which alter starting stats / run behavior. The chosen modifier
applies before the run begins and is shown in the HUD for the whole run.

Satisfies **META-01**: player chooses one of four run modifiers before each run.

## Source-of-truth reconciliation (read before implementing)

CONTEXT.md is the authoritative user-decision source. Where ROADMAP and CONTEXT differ,
follow CONTEXT:

- **Bullet Hell firing:** ROADMAP success criterion 4 phrases it as "sentinel shootInterval
  ÷3 and projDamage ×2", but CONTEXT D-08 specifies "fire 3 shots instead of 1 in
  `fireEnemyShot()`" plus "XP drops ×2 in `dropLoot()`". Implement the **CONTEXT D-08**
  behavior (3 shots per volley via `fireEnemyShot()`, gem value ×2 via `dropLoot()`). The
  net effect — ~3× more incoming enemy fire and 2× XP — satisfies the ROADMAP intent.
- **Glass Cannon regen:** Use the simpler CONTEXT approach (D-06 final bullet): set a
  `player.glassCannonMode` flag and cap regen at 0 inside `updatePlayer()` when the flag is
  set. Do NOT special-case the regen upgrade's `apply()`. This way Vitality/Regeneration
  upgrades can still be picked but produce no healing under Glass Cannon.
