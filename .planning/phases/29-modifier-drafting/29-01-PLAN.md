# Phase 29: Modifier Drafting — Plan 29-01

## Overview

Expand the existing single-pick modifier screen into a two-step draft. Step 1 shows 3 random positive modifiers; Step 2 shows 3 random negative modifiers plus a "Standard Run" skip. The player must pick one positive; the negative pick is optional. Both selections are applied via `applyAndStart()` before the game loop begins. Six positive modifiers (3 from Phase 6 + 3 new) and four negative modifiers form the pools. New runtime effects hook into `updateShooting()` (~line 906), `dropLoot()` (killEnemy area ~line 1344), `update()` (~line 815), and `gainXp()` (~line 1579) using the same flag-and-guard patterns already established by `bulletHellMode` and `player.glassCannonMode`.

### T04 — Rework openModifierSelection() into openModifierDraft() for Step 1

**File:** `games/neon-swarm/game.js`

**Location:** The existing `openModifierSelection()` function (added in Phase 6, near `startGame()` area).

**Changes:**

Rename `openModifierSelection` to `openModifierDraft` (update the function declaration and all call sites — the button event listeners use this name).

Inside the function, change card generation to draw 3 random positive modifiers instead of showing all 4 original modifiers:

```js
function openModifierDraft() {
  initGame();
  gameState = "modifier";
  dom.start.classList.add("hidden");
  dom.gameover.classList.add("hidden");

  // Draw 3 random positive modifiers without replacement
  const pool = [...POSITIVE_MODIFIERS];
  const picks = [];
  for (let i = 0; i < 3; i++) {
    const idx = Math.floor(Math.random() * pool.length);
    picks.push(pool.splice(idx, 1)[0]);
  }

  dom.modifierCards.innerHTML = "";
  picks.forEach((m, i) => {
    const el = document.createElement("div");
    el.className = "upgrade";
    el.style.setProperty("--accent", m.accent);
    el.innerHTML = `<div class="u-icon">${m.icon}</div>
      <div class="u-name">${m.name}</div>
      <div class="u-desc">${m.desc}</div>
      <div class="u-key">${i + 1}</div>`;
    el.addEventListener("click", () => choosePositiveModifier(i));
    dom.modifierCards.appendChild(el);
  });

  // Store the picks so the keyboard handler can reference them
  _currentPositivePicks = picks;

  dom.modifier.classList.remove("hidden");
}
```

Add a module-level variable `let _currentPositivePicks = [];` and `let _currentNegativePicks = [];` near the other `let` globals (they are transient UI state, reset each draft opening).

Also update both button event listeners (they currently call `openModifierSelection`):
