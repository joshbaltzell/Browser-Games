# Phase 17: Enemy Death Splat Trails — Plan

## Overview

This plan adds a `splats` array and four new functions to `game.js` to render color-coded line-segment smears at every enemy death site. The work is purely additive: no existing logic is deleted. The only behavioral change to existing functions is (a) `killEnemy()` gains an optional second parameter and (b) the one callsite inside `resolveBulletHits()` that can supply a killing bullet passes it. All other existing `killEnemy(e)` callsites (in `triggerExplosion`, `applyChainLightning`, `applySplash`, etc.) are left unchanged and the new parameter defaults to undefined, triggering a random-angle fallback.

### T04 — Add `spawnSplats(e, bullet)` helper function

**File:** `games/neon-swarm/game.js`
**Location:** After the closing `}` of `killEnemy()` (~line 1375), before `flushSpawnQueue()` (~line 1377).

Insert the new function:

```js
function spawnSplats(e, bullet) {
  const color = SPLAT_COLORS[e.type] || e.color;
  const count = e.xp >= 4 ? 3 : 1;
  const baseAngle = bullet
    ? Math.atan2(bullet.vy, bullet.vx)
    : Math.random() * TAU;
  const spreadRad = (25 * Math.PI) / 180; // ±25 degrees

  for (let i = 0; i < count; i++) {
    if (splats.length >= SPLAT_CAP) splats.shift();
    const angle = baseAngle + rand(-spreadRad, spreadRad);
    const speed = rand(80, 180);
    const maxLife = rand(0.6, 1.0);
    splats.push({
      x: e.x,
      y: e.y,
      vx: Math.cos(angle) * speed,
      vy: Math.sin(angle) * speed,
      color,
      life: maxLife,
      maxLife,
      thick: true,
    });
  }
}
```

**Note:** `rand()` is already a utility in scope throughout `game.js` (used extensively in `spawnParticles`, `spawnLightningArc`, etc.).

### T07 — Add `updateSplats(dt)` function

**File:** `games/neon-swarm/game.js`
**Location:** After `updateParticles(dt)` (~line 1605), before `spawnFloater`.

```js
function updateSplats(dt) {
  for (const s of splats) {
    s.x += s.vx * dt;
    s.y += s.vy * dt;
    s.vx *= 0.88;
    s.vy *= 0.88;
    s.life -= dt;
  }
  splats = splats.filter((s) => s.life > 0);
}
```

The `0.88` friction per frame decelerates splats so they smear and stop rather than drifting off-screen.

### T09 — Add `drawSplats()` function

**File:** `games/neon-swarm/game.js`
**Location:** After `drawParticles()` (~line 2400), before `drawFloaters()`.

```js
function drawSplats() {
  if (!splats.length) return;
  ctx.save();
  ctx.lineCap = 'round';
  for (const s of splats) {
    const alpha = 0.7 * Math.max(0, s.life / s.maxLife);
    if (alpha <= 0) continue;
    ctx.globalAlpha = alpha;
    ctx.strokeStyle = s.color;
    ctx.shadowColor = s.color;
    ctx.shadowBlur = 8;
    // lineWidth: brute splats are wider; darter splats are thin
    ctx.lineWidth = s.color === SPLAT_COLORS.darter ? 4 : s.color === SPLAT_COLORS.brute ? 7 : 5;
    ctx.beginPath();
    ctx.moveTo(s.x - s.vx * 0.1, s.y - s.vy * 0.1);
    ctx.lineTo(s.x, s.y);
    ctx.stroke();
  }
  ctx.shadowBlur = 0;
  ctx.globalAlpha = 1;
  ctx.restore();
}
```

**Note:** `ctx.shadowBlur = 0` is reset inside the loop's trailing `ctx.restore()` call is at function end — the `ctx.save()`/`ctx.restore()` pair handles full state cleanup. The explicit `ctx.shadowBlur = 0` before `ctx.restore()` is a precaution to avoid leaking shadow blur into subsequent draw calls on the same context state.
