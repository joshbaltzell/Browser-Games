# Phase 26: Brute Stagger Windows — Plan

## Overview

This plan adds a stagger mechanic to Brute enemies in `game.js`. All changes are self-contained: four new fields are added to brute spawn objects, a `triggerStagger()` helper function is added near `killEnemy()` (~line 1344), damage-window tracking and weak-spot multiplier logic are inserted into `updateBullets()` (~line 1194), movement suppression and cooldown decrement are inserted into `updateEnemies()` (~line 1019), and stagger-flash plus weak-spot-pulse rendering is added to the brute draw path inside `render()` (~line 1793). No new files, no new assets, no changes outside `game.js`.

### T03 — Stagger cooldown decrement and movement suppression in updateEnemies()

**File:** `game.js`
**Location:** `updateEnemies()` (~line 1019). Inside the `enemies.forEach(e => { ... })` loop, find the block where enemy position is updated (movement/steering toward player). This is the block to guard.

**Change:** At the top of the per-enemy block (before movement code runs), insert:

```js
// Brute stagger: decrement cooldown
if (e.type === 'brute' && e.staggerCooldown > 0) {
    e.staggerCooldown = Math.max(0, e.staggerCooldown - dt);
}

// Brute stagger: decrement active stagger timer; skip movement while staggered
if (e.type === 'brute' && e.staggerTimer > 0) {
    e.staggerTimer -= dt;
    if (e.staggerTimer <= 0) {
        e.staggerTimer = 0;
        e.weakSpotExposed = false;
    }
    return; // skip movement this frame (use 'continue' if inside a for-loop instead of forEach)
}
```

Note: if the loop uses `for` rather than `forEach`, replace `return` with `continue`.

### T04 — Damage-window tracking and weak-spot multiplier in updateBullets()

**File:** `game.js`
**Location:** `updateBullets()` (~line 1194). Find the section that handles bullet-enemy collision (where `e.hp -= dmg` or equivalent damage application occurs for player bullets hitting enemies). This is inside the nested loop over `bullets` and `enemies`.

**Change:** Directly before the `e.hp -= dmg` line (or equivalent), insert:

```js
// Brute stagger: apply weak-spot multiplier (consume on first hit)
if (e.type === 'brute' && e.weakSpotExposed) {
    dmg *= 4;
    e.weakSpotExposed = false;
}

// Brute stagger: rolling damage window
if (e.type === 'brute' && e.staggerCooldown <= 0) {
    e.damageWindow.push({ dmg: dmg, t: elapsed });
    // Prune entries older than 0.4s
    e.damageWindow = e.damageWindow.filter(entry => elapsed - entry.t <= 0.4);
    // Sum window
    const windowSum = e.damageWindow.reduce((acc, entry) => acc + entry.dmg, 0);
    if (windowSum >= 8) {
        triggerStagger(e);
        e.damageWindow = []; // clear window after triggering
    }
}
```

Important ordering: the weak-spot multiplier must come FIRST so that the boosted damage does not feed back into the window sum for the same frame's stagger check. The window push uses the already-multiplied `dmg` value only if a weak-spot hit is possible — to avoid inflating the next stagger threshold, push the pre-multiplied value instead:

```js
// Corrected ordering — capture raw dmg before weak-spot boost:
let rawDmg = dmg;

if (e.type === 'brute' && e.weakSpotExposed) {
    dmg *= 4;
    e.weakSpotExposed = false;
}

if (e.type === 'brute' && e.staggerCooldown <= 0) {
    e.damageWindow.push({ dmg: rawDmg, t: elapsed });
    e.damageWindow = e.damageWindow.filter(entry => elapsed - entry.t <= 0.4);
    const windowSum = e.damageWindow.reduce((acc, entry) => acc + entry.dmg, 0);
    if (windowSum >= 8) {
        triggerStagger(e);
        e.damageWindow = [];
    }
}
// ... then: e.hp -= dmg;
```

Replace whatever variable name the existing code uses for damage if it is not `dmg` (check the collision block; it may be a literal or named differently).

### T05 — Stagger-flash and weak-spot-pulse rendering

**File:** `game.js`
**Location:** `render()` (~line 1793). Find the section that draws brute enemies — search for the hexagon drawing path for brutes (likely using `ctx.beginPath()` with six sides, or a `drawHexagon`-style helper). It will be inside a block guarded by `e.type === 'brute'` or a switch/if on enemy type.

**Change:** Wrap the existing brute draw call with stagger-flash logic, and add a weak-spot circle after drawing the body:

```js
// --- existing brute draw block (e.type === 'brute') ---

// Stagger flash: override fill color and flicker alpha
let bruteColor = /* existing color variable or literal, e.g. '#ff4444' */;
let bruteAlpha = 1;
if (e.staggerTimer > 0) {
    bruteColor = '#ffffff';
    bruteAlpha = Math.sin(elapsed * 20) > 0 ? 1.0 : 0.35;
}
ctx.save();
ctx.globalAlpha = bruteAlpha;

// ... existing hexagon path and fill using bruteColor instead of the hardcoded color ...

ctx.restore();

// Weak-spot pulse: gold ring drawn on top
if (e.weakSpotExposed) {
    const pulse = Math.sin(elapsed * 12);
    const wsRadius = e.radius * 0.45 + pulse * 2;
    ctx.save();
    ctx.strokeStyle = '#ffd700';
    ctx.lineWidth = 2.5;
    ctx.globalAlpha = 0.85 + pulse * 0.15;
    ctx.beginPath();
    ctx.arc(e.x, e.y, wsRadius, 0, Math.PI * 2);
    ctx.stroke();
    ctx.restore();
}
```

Adapt the color variable to match whatever the existing code uses for the brute's fill. Preserve the existing stroke/outline draw calls; only the fill color and alpha need to change during stagger.
