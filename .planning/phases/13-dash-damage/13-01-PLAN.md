# Phase 13: Momentum Dash Damage — Plan

## Overview

All changes are contained within `executeDash()` (~line 485 in `games/neon-swarm/game.js`). After the existing position teleport and boundary clamp, a point-to-segment distance check sweeps every enemy against the 30px-radius dash cylinder. Hit enemies take damage scaled by the total number clipped in that dash (1×/2×/3+× player.damage), vampirism heals fire per hit, particles spawn at each struck position, and the afterimage trail is widened to radius 30 so the hitbox is visually honest. Enemies that die are collected in a Set, killed via `killEnemy()`, and spliced from `enemies` after the loop. No other functions, globals, or HTML are touched.

### T02 — Add cylinder sweep, damage, vampirism, and particle spawns after boundary clamp in executeDash()

**File:** `games/neon-swarm/game.js`

**Location:** Inside `executeDash()`, immediately after the boundary clamp block (~line 514) and before the `player.invuln = 0.35` line (~line 517). Insert the new block at that gap.

**Read first:** The full `executeDash()` function (lines 485–556) to confirm exact insertion point. Also read `killEnemy()` (~line 1344) to understand its side effects (combo increment, `spawnParticles`, `dropLoot`, `sndKill`) — these must not be duplicated in the new block.

**Code to insert:**

```js
  // ── Phase 13: Dash Damage ─────────────────────────────────────────
  // Point-to-segment distance for each enemy against the dash cylinder.
  // Segment: (ox,oy) → (player.x, player.y). Cylinder radius: 30 + e.radius.
  {
    const segDx = player.x - ox;
    const segDy = player.y - oy;
    const segLenSq = segDx * segDx + segDy * segDy;

    const clipped = [];
    for (const e of enemies) {
      if (e.hp <= 0) continue;
      // Project enemy center onto segment; clamp t to [0,1]
      const t = segLenSq > 0
        ? Math.max(0, Math.min(1, ((e.x - ox) * segDx + (e.y - oy) * segDy) / segLenSq))
        : 0;
      const closestX = ox + t * segDx;
      const closestY = oy + t * segDy;
      const dist = Math.hypot(e.x - closestX, e.y - closestY);
      if (dist < 30 + e.radius) {
        clipped.push(e);
      }
    }

    if (clipped.length > 0) {
      const mult = clipped.length >= 3 ? 3.0 : clipped.length === 2 ? 2.0 : 1.0;
      const dashDamage = player.damage * mult;
      const toKill = new Set();

      for (const e of clipped) {
        e.hp -= dashDamage;
        spawnParticles(e.x, e.y, e.color, 8, [40, 160]);
        if (player.lifesteal > 0) {
          player.hp = Math.min(player.maxHp, player.hp + player.lifesteal);
        }
        if (e.hp <= 0) {
          toKill.add(e);
        }
      }

      for (const e of toKill) {
        killEnemy(e);
      }
      if (toKill.size > 0) {
        enemies = enemies.filter(e => !toKill.has(e));
      }
    }
  }
  // ── End Phase 13 ─────────────────────────────────────────────────
```

**Exact insertion point:** Place this block after the closing brace of the boundary clamp (after `player.y = Math.max(player.radius, Math.min(H - player.radius, player.y));` at ~line 514) and before `player.invuln = 0.35;` (~line 517).

**Key decisions explained:**

- `if (e.hp <= 0) continue` at the top of the sweep loop skips any enemy already dead from a prior frame — avoids ghost hits.
- `segLenSq > 0` guard handles the theoretical edge case where the dash was fully boundary-clamped to zero distance (player already at corner); without it, division by zero would produce `NaN` for `t`.
- `toKill` Set + post-loop `enemies.filter()` mirrors the removal pattern in `updateBullets()` (~line 1194) — never mutate `enemies` while iterating it.
- `killEnemy(e)` is called for each killed enemy: it already calls `spawnParticles`, `sndKill`, `dropLoot`, increments `combo` and `kills`, and handles all build-specific kill callbacks (Blood Surge, Death Dance, Soul Harvest). Do not replicate any of those calls in this block.
- Vampirism heal (`player.lifesteal`) fires once per hit regardless of kill outcome, matching the `killEnemy()` pattern at ~line 1352 where lifesteal is conditioned on `player.lifesteal > 0`.
- `mult` is computed from `clipped.length` (total enemies in cylinder), applied uniformly to every hit in that dash. This is intentional: a tight cluster rewards the player with high per-enemy damage, not just high total damage.
