# Phase 20: Low HP Heartbeat Pulse — Plan

## Overview

This phase adds a low-HP danger signal to game.js: two new globals track the heartbeat accumulator, `initGame()` resets them, the `update()` loop drives the beat timer and fires audio, a new `playHeartbeat()` function synthesizes a 55 Hz thump, and `render()` draws a radial red pulse from screen edges each frame. Total surface area is small (~25 lines across four touch points in one file).

### T03 — Add playHeartbeat() audio function

**File:** game.js  
**Location:** Near other Web Audio synthesis helpers. A safe anchor is just before or after the `triggerExplosion()` function (~line 1178), or wherever `audioCtx` is used to create oscillators/gain nodes elsewhere in the file. Place the full function as a standalone named function declaration.

**Add:**
```js
function playHeartbeat() {
  if (!audioCtx || muted) return;
  const osc = audioCtx.createOscillator();
  const gain = audioCtx.createGain();
  osc.connect(gain);
  gain.connect(audioCtx.destination);
  osc.type = 'sine';
  osc.frequency.setValueAtTime(55, audioCtx.currentTime);
  // hpRatio: 1.0 at 30% HP, 0.0 near death — recomputed here for audio gain
  const hpRatio = Math.max(0, player.hp / (player.maxHp * 0.3));
  gain.gain.setValueAtTime(0.12 * (1 - hpRatio), audioCtx.currentTime);
  gain.gain.exponentialRampToValueAtTime(0.0001, audioCtx.currentTime + 0.08);
  osc.start(audioCtx.currentTime);
  osc.stop(audioCtx.currentTime + 0.08);
}
```

### T04 — Drive heartbeat in update()

**File:** game.js  
**Location:** `update(rawDt)` (~line 815). Find the section that handles `gameState === 'playing'` logic. A good anchor is right after the Death Dance / berserker state checks (which also inspect `player.hp`) and before `updatePlayer(dt)` is called (~line 883). Insert the heartbeat accumulator block there.

**Add:**
```js
// Heartbeat pulse — active below 30% HP
if (player.hp < player.maxHp * 0.3) {
  const hpRatio = player.hp / (player.maxHp * 0.3); // 1.0 at 30%, ~0 near death
  heartbeatPeriod = 0.35 + hpRatio * 0.85;           // lerp 0.35s–1.2s
  heartbeatPhase += dt;
  if (heartbeatPhase >= heartbeatPeriod) {
    heartbeatPhase = 0;
    playHeartbeat();
  }
} else {
  heartbeatPhase = 0; // reset so next entry starts a clean cycle
}
```

### T05 — Draw pulse overlay in render()

**File:** game.js  
**Location:** `render()` (~line 1793). Scroll to the very end of the render function, after all existing overlay draws (Death Dance vignette, overdrive flash, level-up flash, shake, etc.) and before the closing brace of `render()`. The heartbeat overlay must be the last (or near-last) canvas draw so it composites on top of all game elements.

**Add:**
```js
// Heartbeat pulse overlay
if (gameState === 'playing' && player.hp < player.maxHp * 0.3) {
  const hpRatio = player.hp / (player.maxHp * 0.3);
  const pulseIntensity = Math.max(0, Math.sin(heartbeatPhase / heartbeatPeriod * Math.PI)) * 0.18;
  if (pulseIntensity > 0) {
    const cx = canvas.width / 2;
    const cy = canvas.height / 2;
    const innerR = canvas.height * 0.3;
    const outerR = Math.sqrt(cx * cx + cy * cy); // corner distance — covers full screen
    const hbGrad = ctx.createRadialGradient(cx, cy, innerR, cx, cy, outerR);
    hbGrad.addColorStop(0, 'rgba(180,0,0,0)');
    hbGrad.addColorStop(1, `rgba(180,0,0,${pulseIntensity.toFixed(3)})`);
    ctx.save();
    ctx.fillStyle = hbGrad;
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    ctx.restore();
  }
}
```

## Verification

Run the game and confirm each item from the context checklist:

1. Drop to 28% HP — edge pulse appears within 1.2s and thumps are audible (if unmuted).
2. Drop to ~5% HP — beat visibly faster than at 28%; edges bleed more intensely at pulse peak.
3. Center of screen stays clear — inner radius gap keeps the playfield readable.
4. Heal above 30% (soul harvest, regen tick, lifesteal) — pulse stops immediately.
5. Toggle mute button — audio stops, visual continues.
6. Activate freeze powerup at low HP — heartbeat slows with game time.
7. Check browser console — no errors from `playHeartbeat()` before first user gesture (audioCtx null guard).
8. Start a new run from game over — no residual pulse on the start screen.
9. Die with Death Dance active — both edge pulse and central Death Dance vignette visible simultaneously.
10. Verify `pulseIntensity` never exceeds 0.18 (sine peak * 0.18) by logging or visual inspection.
