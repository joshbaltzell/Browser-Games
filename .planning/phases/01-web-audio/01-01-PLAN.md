# Phase 01 — Web Audio — Plan 01-01

Add synthesized sound effects to Neon Swarm using the Web Audio API. Every significant
game event (shoot, hit, kill, level-up, bomb, freeze, overdrive, player hit) plays a
short programmatic oscillator tone. No external assets, no `fetch()`, no `<audio>`
elements — must run from `file://`. All changes are confined to
`games/neon-swarm/game.js`; `index.html` is NOT modified (no DOM changes needed).

This plan is a single wave of three sequential tasks:
- T01 — Audio engine: global `audioCtx`, lazy init/unlock, and the shared `playTone()` helper.
- T02 — Sound functions: one named function per event (D-05 … D-12) built on the helper.
- T03 — Integration: wire each sound function into its game-event call site (D-13 … D-20).

       section (after `unlockAudio`). It is the single primitive all sound functions build on.
       Behavior:
       - GUARD: if `audioCtx` is null, return immediately (silently no-op). This makes every
         sound a safe no-op when audio is unavailable or not yet unlocked.
       - Also call `audioCtx.resume()` defensively at the start (D-02) — wrap in a try/catch
         or check state so a rejected promise never throws synchronously. (`resume()` returns
         a promise; do not await it.)
       - Read parameters from `opts` with sensible defaults:
         - `type` (oscillator type: "sine" | "triangle" | "sawtooth" | "square"), default "sine"
         - `freq` (start frequency in Hz), default 440

           over the duration using `frequency.exponentialRampToValueAtTime`, falling back to
           `linearRampToValueAtTime` when either endpoint is <= 0 since exponential ramps
           require strictly positive values — e.g. the bomb sweeps 60Hz->20Hz, both positive,
           but keep the guard for safety)

         - `gain` (peak gain, max 0.3 per D-21), default 0.1

         `audioCtx.createGain()`. Connect osc -> gain -> `audioCtx.destination`.
       - Use `const now = audioCtx.currentTime;` as the time base.
       - Set `osc.type = type` and `osc.frequency.setValueAtTime(freq, now)`. If `endFreq`
         is given, schedule the ramp to `now + dur`.
       - GAIN ENVELOPE (D-22 — short, auto-disposing, no clicks): start gain near zero
         (`gain.gain.setValueAtTime(0.0001, now)`), a quick attack ramp up to `gain` peak
         (e.g. `linearRampToValueAtTime(gain, now + 0.005)`), then a decay to a tiny value
         (`exponentialRampToValueAtTime(0.0001, now + dur)`). This prevents clicks/pops.
       - Schedule playback: `osc.start(now)` and `osc.stop(now + dur)`. Because each call
         creates fresh nodes and stops them with an exact time, the nodes are garbage-collected
         automatically — NO persistent nodes, NO node pooling (D-22).
       - Wrap the node creation/scheduling in a try/catch so any unexpected runtime error in
         one tone never breaks the game loop.
