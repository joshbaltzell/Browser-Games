# Phase 04 ‚Äî Power-up Timer Display ‚Äî Plan 01

## Goal

Show a visible countdown label and depleting bar for active Freeze and Overdrive
power-ups so the player always knows how long their power-up lasts. This is a
render-only change: add two duration constants and one new draw function,
`drawPowerupTimers()`, called from `render()` after the shake transform ends. No
game-logic changes ‚Äî `freezeTimer` and `overdriveTimer` already exist and are
already counted down in `update()`.

## Context Summary

- `freezeTimer` (global, line ~92) is set to `3.0` by `activateFreeze()` (line ~680)
  and decremented in `update()` (line ~333). Freeze active iff `freezeTimer > 0`.
- `overdriveTimer` (global, line ~93) is set to `5.0` by `activateOverdrive()`
  (line ~683/696) and decremented in `update()` (line ~334). Overdrive active iff
  `overdriveTimer > 0`.
- `render()` (line ~760) draws the world inside a `ctx.save()`/`ctx.restore()`
  shake transform (lines 764‚Äì779). Overlay draws (`drawFloaters()` line 781,
  `drawCombo()` line 799) happen AFTER `ctx.restore()` so they don't jitter.
- `drawCombo()` (lines 1034‚Äì1059) is the structural reference: it draws centered
  monospace text plus a background+fill bar at a fixed screen position using
  `ctx.save()`/`ctx.restore()`.
- `COMBO_DECAY` constant is at line 24; new duration constants go adjacent to it.
- `POWERUP_TYPES` (lines 37‚Äì41) already defines `freeze` icon `‚ùÑÔ∏è` and `overdrive`
  icon `‚ö°` ‚Äî but CONTEXT specifies exact display colors that differ from the
  collectible `overdrive.color` (`#00e5ff`), so the timer colors are defined in
  the draw function, not read from `POWERUP_TYPES`.
