# Phase 02 — Plan 01: Enemy Shape Vocabulary

## Goal

Replace the uniform circle rendering for all enemy types in `drawEnemies()` with
distinct geometric silhouettes — one shape per enemy type — so the player can read
the battlefield at a glance. All shapes keep the existing neon glow aesthetic, flash
behavior, sentinel core dot, and HP bar. No game logic, no collision footprint changes.

Shape map (per CONTEXT.md decisions):
- Chasers → circle (baseline, D-01)
- Darters → equilateral triangle, leading vertex pointed at player (D-02, D-13)
- Brutes → regular hexagon, thick/solid (D-03)
- Sentinels → diamond (square rotated 45°) + existing white core dot kept (D-04)
- Spores → irregular ~5-point blob (lumpy pentagon) (D-05)
- Sporelings → small circle (D-06)

       global ctx.) (D-03)

       top/right/bottom/left: (x, y-r), (x+r, y), (x, y+r), (x-r, y). All vertices sit on the
       radius-r circle so the collision footprint is unchanged (D-04, D-11).

    4. `drawBlob(ctx, x, y, r)` — irregular ~5-point lumpy pentagon (D-05). Use a fixed vertex

       The variation must be a constant array (not Math.random) so the shape does not jitter
       frame-to-frame. Plot vertex i at `(x + rr*cos(a), y + rr*sin(a))`.

    All four end with `ctx.closePath()`. Do not modify drawHexagon itself.
  </action>
