# Phase 31: Corruptor Enemy — Plan

### T02 — Draw Corruptor Shape in render()

**File:** `game.js`
**Location:** Inside `render()` (~line 1793), within the per-enemy loop that draws enemy shapes. The existing code already branches on `e.type` (e.g., to draw triangle for darter, hexagon for brute, diamond for sentinel). Add a new `else if (e.type === 'corruptor')` branch.

**Add this drawing block:**
```js
} else if (e.type === 'corruptor') {
  // Inverted triangle (point-down)
  ctx.save();
  ctx.translate(e.x, e.y);
  ctx.strokeStyle = e.color || '#fd79a8';
  ctx.fillStyle = (e.color || '#fd79a8') + '33'; // translucent fill
  ctx.lineWidth = 2;
  ctx.beginPath();
  const r = e.radius;
  // Three vertices: top-left, top-right, bottom-center (inverted)
  ctx.moveTo(-r, -r * 0.8);
  ctx.lineTo( r, -r * 0.8);
  ctx.lineTo( 0,  r * 1.0);
  ctx.closePath();
  ctx.fill();
  ctx.stroke();
  // Spiral / chaos symbol inside
  ctx.strokeStyle = '#fd79a8';
  ctx.lineWidth = 1.2;
  ctx.beginPath();
  // Small spiral: two arcs approximating a chaos symbol
  for (let i = 0; i <= 48; i++) {
    const angle = (i / 48) * Math.PI * 3.5; // 1.75 full turns
    const rad = (i / 48) * (r * 0.55);
    const sx = Math.cos(angle) * rad;
    const sy = Math.sin(angle) * rad;
    if (i === 0) ctx.moveTo(sx, sy);
    else ctx.lineTo(sx, sy);
  }
  ctx.stroke();
  ctx.restore();
}
```

### T03 — Trigger Gem Corruption in killEnemy()

**File:** `game.js`
**Location:** Inside `killEnemy(e)` (~line 1344), after the standard kill processing (XP gain, particle spawn, etc.). Add the following block.

**Add this block at the end of killEnemy, before or after existing type-specific checks:**
```js
// Corruptor death: corrupt all on-screen gems
if (e.type === 'corruptor') {
  for (let i = 0; i < gems.length; i++) {
    const g = gems[i];
    if (g.corrupted) continue; // do not double-corrupt
    g.corrupted = true;
    g.value = Math.max(1, Math.floor(g.value * 0.5));
    g.color = '#e17055';
    g.corruptedTimer = 5.0;
    // Compute flee vector away from player
    const dx = g.x - player.x;
    const dy = g.y - player.y;
    const dist = Math.sqrt(dx * dx + dy * dy) || 1;
    g.vx = (dx / dist) * 80;
    g.vy = (dy / dist) * 80;
  }
  spawnFloater(e.x, e.y - 20, 'CORRUPTION!', '#fd79a8', 20);
}
```

`spawnFloater` is confirmed in the architecture reference (used elsewhere in the codebase for damage numbers and event text). `gems` is a confirmed global array reset in `initGame`.

### T04 — Move Fleeing Gems in update()

**File:** `game.js`
**Location:** Inside `update(rawDt)` (~line 815), in the gem update loop. Gems are already iterated here to handle pickup proximity and movement toward the player. Extend the per-gem iteration with a corruption movement block.

Find the loop that iterates `gems` (checking distance to player for pickup). Inside that loop, add the following logic **before** the pickup distance check so fleeing gems move away first:

```js
// --- Corruption flee movement ---
if (g.corrupted && g.corruptedTimer > 0) {
  g.corruptedTimer -= dt;
  g.x += g.vx * dt;
  g.y += g.vy * dt;
  if (g.corruptedTimer <= 0) {
    g.vx = 0;
    g.vy = 0;
  }
  // Remove gem if it has fled off-screen
  const MARGIN = 50;
  if (g.x < -MARGIN || g.x > canvas.width + MARGIN ||
      g.y < -MARGIN || g.y > canvas.height + MARGIN) {
    gems.splice(i, 1);
    i--;
    continue;
  }
}
// --- End corruption flee movement ---
```

Note: The gem loop uses an index-based `for` loop with `gems.splice` already (gems are removed on pickup). Use `i--` after splice to keep the index correct, consistent with the existing pattern in `updateBullets` (~line 1194) and `updateEBullets` (~line 1145).
