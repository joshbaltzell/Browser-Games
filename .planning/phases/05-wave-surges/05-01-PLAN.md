# Phase 05 — Wave Surge Announcements — Plan 05-01

## Goal

Add periodic, pre-announced enemy surge events so the run has a rhythm of
anticipation → burst → relief. After 60s of play, a surge type is selected,
announced with a 3-second centered on-canvas text warning, then a burst of that
enemy type spawns through the existing `spawnQueue`. Normal spawning continues
in parallel; no existing spawn logic is altered.

All work happens in `games/neon-swarm/game.js`. No DOM changes (the announcement
renders on the canvas), so `index.html` is untouched. Game must continue to run
from `file://` with no build step and no dependencies.

## Decision Resolutions

- **D-13 (alpha fade):** The CONTEXT.md text walks back the complex formula to a
  simpler rule. This plan implements the simpler rule explicitly: announcement
  text is fully solid (alpha = 1) for the first 2.5s of the 3.0s warning, then
  fades linearly to 0 over the final 0.5s. Formula:
  `alpha = surgeWarningTimer >= 0.5 ? 1 : surgeWarningTimer / 0.5`
  (since `surgeWarningTimer` counts down from 3.0 to 0).
- **Claude's discretion (screen-edge pulse):** A subtle full-screen tint pulse
  during the warning IS included (low cost, strong "telegraph" feel), at low
  alpha tied to the surge color. Implemented in render alongside the text.
- **Claude's discretion (slow-mo at burst):** NOT included — a slow-mo at every
  surge would over-trigger and clash with the existing hit-stop/bomb slow-mo
  cadence. Left out to keep pacing readable.
- **Surge counts:** Starting values from D-05 are kept (4 brutes, 10 darters,
  2 sentinels, 3 spores).

   "BRUTE SURGE!" in purple precedes 4 brutes; "DARTER STORM!" in orange precedes
   10 darters. Color matches the enemy type.
4. **Parallel spawning:** During the 3s warning, confirm normal enemies keep
   spawning (the field is not frozen or paused).
5. **Type gating:** Set `elapsed = 70` in console and trigger a surge repeatedly
   (`surgeState="idle"; surgeTimer=0;`) — only DARTER STORM should ever be chosen
   (brute=90, spore=120, sentinel=150 not yet unlocked). At `elapsed = 200`, all
   four types are eligible.
6. **Cooldown rhythm:** After a surge completes, confirm `surgeTimer` resets to a
   value between 30 and 50 (log it) and the next surge follows after that delay.
7. **Cap safety:** Trigger a darter storm while the field is already crowded;
   confirm `enemies.length` does not blow far past 200 (surge respects the cap).
8. **No regressions:** Confirm shooting, leveling, power-ups, freeze tint, combo
   readout, and screen shake all still work, and there are no console errors.
