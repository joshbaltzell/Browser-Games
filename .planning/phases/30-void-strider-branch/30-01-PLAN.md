# Phase 30: Void Strider Branch — Plan

## Overview

This plan adds the VOID STRIDER skill branch and VOID DANCER fusion to Neon Swarm in six ordered tasks. T01 lays the data foundation (player flags, SKILL_TREE entry, FUSION_SKILLS entry). T02 extends the CSS grid to 8 columns. T03 captures pre-dash position in `executeDash()` and adds the Void Step travel-path damage. T04 adds the Shockwave arrival explosion and the Singularity pull-plus-triple-blast block, with correct mutual-exclusion guarding. T05 wires Void Dancer into the same arrival block and resets `shootTimer`. T06 implements Afterburn — extending afterimage spawn, `updateAfterimages()`, and `updateEnemies()` with contact-damage logic. Tasks build on one another in dependency order; each targets a single, well-bounded location.

### T04 — Add Shockwave arrival explosion and Singularity pull+blast in `executeDash()`

**File:** `game.js`

**Location:** `executeDash()` (~line 485). Insert the following block **after** the Void Step block added in T03. The order of all arrival effects is: Void Step → Singularity (if owned) → Shockwave single-call (if owned without Singularity) → Void Dancer (T05 adds this last).

```js
// Singularity: pull enemies within 100px toward landing, then Shockwave × 3
if (player.vsSingularity) {
  for (const e of enemies) {
    const dist = Math.hypot(e.x - player.x, e.y - player.y);
    if (dist < 100 && dist > 0) {
      const ux = (player.x - e.x) / dist;
      const uy = (player.y - e.y) / dist;
      e.x += ux * 200 * 0.15;
      e.y += uy * 200 * 0.15;
      // Clamp pulled enemy to canvas bounds
      e.x = Math.max(e.radius, Math.min(W - e.radius, e.x));
      e.y = Math.max(e.radius, Math.min(H - e.radius, e.y));
    }
  }
  triggerExplosion(player.x, player.y, 60, player.damage * 1.5);
  triggerExplosion(player.x, player.y, 60, player.damage * 1.5);
  triggerExplosion(player.x, player.y, 60, player.damage * 1.5);
}

// Shockwave (single): only when vsShockwave owned without vsSingularity
if (player.vsShockwave && !player.vsSingularity) {
  triggerExplosion(player.x, player.y, 60, player.damage * 1.5);
}
```

`triggerExplosion(x, y, radius, damage)` is at ~line 1178. `W` and `H` are the canvas width/height globals. Three separate `triggerExplosion` calls produce visually distinct triple particle bursts; the combined effective damage totals `player.damage * 4.5` across the three blasts.

  for (const a of afterimages) {
    if (!a.damageActive) continue;
