# Phase 21: Damage Number Floaters — Plan

## Overview

This phase adds a `dmgNum` floater subtype that spawns at every damage event — bullet hits, splash hits, and chain lightning hops — and renders with per-source color coding, magnitude-scaled font size, and a final-40%-only alpha fade. The existing `spawnFloater` / `drawFloaters` infrastructure in `game.js` is extended rather than replaced: a new helper `spawnDmgNum()` manages its own 40-item cap and produces objects with `type:'dmgNum'` to opt into distinct update and render behavior. The primary bullet-hit floater in `resolveBulletHits()` (which already spawns a text damage number via `spawnFloater`) is replaced with `spawnDmgNum()` to consolidate all damage number display under one system.

### T01 — Add `spawnDmgNum()` helper

**File:** `games/neon-swarm/game.js`
**Location:** Insert immediately after `spawnBuildFloater()` (line ~1639, just before `openSkillTree()` at line 1641).

**Code to add:**

```js
// Damage number floater (Phase 21). Separate 40-item cap; slower drift; alpha
// fades only in the final 40% of life so numbers stay crisp then disappear.
function spawnDmgNum(x, y, dmg, color) {
  const MAX_DMG_NUMS = 40;
  const size = Math.min(24, Math.max(11, 10 + dmg * 0.4));
  const maxLife = 0.9;
  // Evict oldest if at cap
  const existing = floaters.filter(f => f.type === 'dmgNum');
  if (existing.length >= MAX_DMG_NUMS) {
    const idx = floaters.indexOf(existing[0]);
    if (idx !== -1) floaters.splice(idx, 1);
  }
  floaters.push({
    type: 'dmgNum',
    x: x + rand(-6, 6),
    y: y - 10,
    text: Math.round(dmg),
    color,
    size,
    vy: -35,
    vx: rand(-8, 8),
    life: maxLife,
    maxLife,
  });
}
```

### T06 — Update `updateFloaters()` to support `dmgNum` alpha-fade logic

**File:** `games/neon-swarm/game.js`
**Location:** `updateFloaters()` at line ~1624. The current alpha line is:

```js
    f.alpha = Math.max(0, f.life / f.maxLife);
```

**Replace with:**

```js
    if (f.type === 'dmgNum') {
      // Stay fully opaque until the final 40% of life, then fade to 0
      const fadeStart = f.maxLife * 0.4;
      f.alpha = f.life <= fadeStart ? f.life / fadeStart : 1;
    } else {
      f.alpha = Math.max(0, f.life / f.maxLife);
    }
```

Note: `dmgNum` floaters use `vy: -35` (constant drift, no deceleration). The existing `f.vy *= 0.94` deceleration applies to all floaters — that is acceptable for damage numbers; they slow slightly as they rise, which reads naturally. No change needed to velocity damping.

Run the game at `file://` by opening `games/neon-swarm/index.html`. Confirm:

1. **Normal hit** — shooting a chaser produces a white number at the enemy that rises slowly and fades out over ~0.9 s; no abrupt pop-off.
2. **Crit hit** — with `critChance > 0` (Ghost branch / crits skill), a critical hit produces a pale-yellow (`#fffb96`) number visibly larger than a same-target non-crit.
3. **Splash** — with the Explosives upgrade active, hitting a group shows orange numbers on each enemy caught in the blast, distinct from the white number on the direct-hit enemy.
4. **Chain lightning** — with Chain Lightning fusion skill, each hop after the primary hit shows a cyan number at the chained target; the primary hit still shows white/yellow.
5. **Bomb powerup** — activating bomb (`triggerExplosion` call path) produces orange numbers on all enemies in range.
6. **Font size scaling** — a 20+ damage hit renders a larger number than a 2-damage hit; visually clear at a glance.
7. **Alpha fade** — numbers remain fully opaque for the first ~60% of their life, then fade cleanly; no hard disappearance.
8. **40-floater cap** — during overdrive + chain lightning on a dense pack, open the browser console, add `console.log(floaters.filter(f=>f.type==='dmgNum').length)` temporarily; count never exceeds 40.
9. **No orphaned floaters on reset** — starting a new game after death shows no leftover numbers; the `floaters` array is cleared by `initGame()`.
10. **No frame drops** — frame rate stays smooth (>55 fps) during maximum AOE + chain lightning; use browser performance tab to confirm no GC spikes from floater allocation.
