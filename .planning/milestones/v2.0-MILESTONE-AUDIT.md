## Bugs Found and Fixed at Audit

Four bugs discovered by integration checker and fixed inline before audit completion:

1. **`sndLevelUp()` never called** — `gainXp()` at line 1592 did not call `sndLevelUp()`. Added call, completing AUDIO-01 (level-up was the only missing sound event).

2. **`spawnFloater` argument order inverted** — `gainXp` passed `(text, x, y, ...)` but signature is `(x, y, text, ...)`. Level-up floater rendered at wrong position with wrong text content. Fixed argument order.

3. **Mute button unclickable** — `#hud { pointer-events: none }` cascaded to `#mute-btn`, making the click handler unreachable. Added `pointer-events: auto` to `#mute-btn` in style.css.

4. **`applyHeadstart()` skipped `checkBuildName()`** — Headstart modifier applied skill nodes without running build-name detection. Build names were silently missed at game start for Headstart runs. Added `checkBuildName()` call at end of `applyHeadstart()`.

## Tech Debt

### Dead Boolean Flags
`player.phaseRunner`, `player.fullAuto`, `player.overclockedBurst` are set by skill `apply()` functions but never read — all runtime logic uses `player.ownedSkills.has(id)` instead. Harmless dead code.

### Missing Verification Documents
Phases 02–04, 07–10 lack `VERIFICATION.md` files. All features were confirmed by code inspection and integration checker. Creating formal verification docs for these phases is recommended before starting v3.0 work.

### No Nyquist Coverage
No `VALIDATION.md` files exist for any phase. The game is a canvas-only browser game with no automated test harness; all behavioral verification requires human play. This is acceptable for a file:// game but should be noted for future milestone planning.
