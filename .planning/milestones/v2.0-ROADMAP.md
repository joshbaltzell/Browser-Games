## Overview

Ten focused phases to transform Neon Swarm from a working prototype into a polished,
replayable game. Each phase is a self-contained feature addition to `games/neon-swarm/game.js`.
All phases are independent — no inter-phase dependencies.

## Milestone Summary

**Key Decisions:**
- Web Audio via AudioContext (not HTML5 Audio) — file:// compatible, programmable synthesis
- Enemy shapes via Canvas path drawing (not sprites) — no assets, matches existing glowCircle
- Modifier cards reuse `.upgrade-cards` CSS class (Phase 11 removal required restoration)
- Headstart uses random shuffle of SKILL_TREE nodes (not hardcoded — fixed at audit)
- Last Stand implemented as fusion skill requiring cross-branch prerequisites
- Dash triggered by Shift key — natural muscle memory for "burst move"
- Build names shown as floaters + persistent HUD text (two-layer feedback)

**Issues Resolved:**
- sndLevelUp() defined but never called — wired at audit
- spawnFloater() arg order inverted in gainXp — fixed at audit
- Mute button unclickable (pointer-events cascade) — fixed at audit
- applyHeadstart() skipped checkBuildName() — fixed at audit
- Phase 6 CSS gap (Phase 11 removed .upgrade-cards) — restored at audit

**Issues Deferred:**
- Missing VERIFICATION.md for phases 02-04, 07-10 (browser game — no automated test harness)
- Dead boolean flags (player.phaseRunner, player.fullAuto, etc.) — harmless, low priority

**Technical Debt:**
- Some player flags set by skill apply() but runtime logic uses ownedSkills.has() directly — minor inconsistency
